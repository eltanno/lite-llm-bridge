# Quick Task 260522-s9z

## Objective

Replace the 3-model test config (260522-i4q) with a real-name **catalog** of 9 models, and add a
project-local launcher that wires Claude Code's opus/sonnet/haiku tiers to catalog models via
`ANTHROPIC_DEFAULT_*_MODEL`. Swapping a tier becomes a client env-var change (no container restart).
Isolated — no `~/.bashrc` / global changes.

## Catalog (all cloud-verified against the live /v1/models)

- Anthropic (Max OAuth forwarding, scoped): `claude-opus-4-7`, `claude-sonnet-4-6`, `claude-haiku-4-5-20251001`
- Ollama Cloud: `deepseek-v4-pro`, `deepseek-v4-flash`, `kimi-k2.6`, `qwen3.5:397b`, `glm-5.1`, `minimax-m2.7`

(qwen3.6 excluded — library/local-only, not cloud-hosted. Per-model thinking tunable via `reasoning_effort`.)

## Tasks

1. `config.yaml` — real-name catalog + `model_group_settings` forwarding scoped to the 3 Anthropic models.
2. `bin/claude-gw` — isolated launcher: loads master key from `.env`, sets gateway env + a tier mapping, execs `claude`.
3. Recreate gateway; verify all 9 `model_name`s route: 6 Ollama via direct curl, 3 Anthropic via `claude -p` (Max OAuth).

## must_haves

- All 9 catalog `model_name`s resolve through the gateway (HTTP 200 / successful completion).
- Anthropic models route via the forwarded Max bearer; Ollama via `OLLAMA_API_KEY` (bearer never leaks).
- The launcher sets gateway env in its own process only (no global mutation).

## Notes

- Tier swapping = edit the mapping in `bin/claude-gw` or override its env vars; no restart.
- Catalog additions = edit `config.yaml` + `docker compose up -d --force-recreate litellm`.
