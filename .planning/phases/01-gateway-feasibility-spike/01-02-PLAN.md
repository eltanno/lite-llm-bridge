<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Claude Code (host) → LiteLLM (localhost:4000) | Master key (x-litellm-api-key) and Max OAuth bearer (Authorization) cross here |
| LiteLLM → api.anthropic.com | The forwarded Max OAuth bearer crosses here — and MUST NOT cross toward Ollama |
| Human ↔ Claude Max login | OAuth credential obtained at runtime via browser; stored by Claude Code (~/.claude/.credentials.json), never an env var |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing / Repudiation | Gateway auth | mitigate | Master key carried in x-litellm-api-key header; LITELLM_MASTER_KEY enforces request auth (CC-01) |
| T-02-02 | Information Disclosure | Auth-header collision | mitigate | Use ANTHROPIC_CUSTOM_HEADERS (not ANTHROPIC_AUTH_TOKEN) so the LiteLLM key and the OAuth bearer ride in separate headers; Task 1 asserts both ANTHROPIC_API_KEY and ANTHROPIC_AUTH_TOKEN are unset (Pitfall 1/2) |
| T-02-03 | Information Disclosure | OAuth bearer egress | mitigate | E-01 confirms forwarding is scoped to Claude groups; the safe fallback (global forwarding) is harmless because no Anthropic API key lives in the container, so a bearer reaching Ollama is rejected (D-03) |
| T-02-04 | Elevation of Privilege / Billing bypass | ANTHROPIC_API_KEY in host env | mitigate | Task 1 unsets it and verifies; /status check in Task 2 confirms subscription auth, not API key, is in use (Pitfall 2) |
| T-02-05 | Information Disclosure | Decision-gate fallback secret | mitigate | If the API-key fallback is taken, the key goes only to gitignored .env via os.environ/; Task 3 leak gate `grep` for sk-ant- prevents committing it (GW-03) |
</threat_model>
