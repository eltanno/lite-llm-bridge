## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Claude Code (host) → LiteLLM (localhost:4000) | The haiku-alias request carries the gateway master key |
| LiteLLM → https://ollama.com/v1 | OLLAMA_API_KEY Bearer crosses here; the Anthropic OAuth bearer and anthropic-beta headers MUST NOT cross here |
| Ollama model ↔ host filesystem (via tool use) | Tool_use/tool_result drive real file writes and bash on the host during the spike |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | Anthropic bearer / beta headers reaching Ollama | mitigate | Forwarding scoped to Claude groups (Plan 02 E-01) so the Ollama group gets no forwarded bearer; CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1 client-side; Task 1 log check confirms no anthropic-beta on Ollama-bound requests (HDR-01) |
| T-03-02 | Tampering | Anthropic-specific params breaking/over-driving Ollama | mitigate | drop_params: true strips unknown params before the Ollama call (D-05); validated by the agentic session completing, not by curl (Pitfall 7) |
| T-03-03 | Information Disclosure | OLLAMA_API_KEY in committed files | mitigate | Key only in gitignored .env via os.environ/; Task 3 leak gate greps config.yaml/README (GW-03) |
| T-03-04 | Tampering | Untrusted model output driving host tool use | accept | Single-user local PoC; the developer initiates and supervises the spike in their own working tree; production sandboxing is out of v1 scope |
| T-03-05 | Spoofing | Wrong Ollama endpoint (missing /v1) silently failing | mitigate | E-02 (Plan 01) validated the exact api_base; Task 1 re-asserts openai/ + /v1 + :cloud before the spike (Pitfall 3/4) |
</threat_model>
