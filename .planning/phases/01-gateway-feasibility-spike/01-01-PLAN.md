<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Claude Code (host) → LiteLLM (localhost:4000) | Incoming requests carry the gateway master key and the Max OAuth bearer; only loopback should reach this port |
| LiteLLM container → Anthropic / Ollama Cloud (egress) | Outbound credentials cross here; the Anthropic OAuth bearer must reach ONLY Anthropic, never Ollama |
| Repo (committed files) → public/git | compose.yaml/config.yaml/.env.example are committed; .env must never cross this boundary |
| Docker image registry → container | The pinned image is executable code pulled into the trust zone |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | Committed config/compose/docs | mitigate | Secrets only in gitignored .env via os.environ/ refs; Task 1 + Task 2 + Task 3 each run a `grep -RInE 'sk-[0-9a-f]{16,}'` leak gate; .env.example holds placeholders only (GW-03) |
| T-01-02 | Tampering / Supply chain | LiteLLM Docker image | mitigate | Pin `ghcr.io/berriai/litellm:main-stable`; compose.yaml verify gate rejects `:latest` and `1.82.7`/`1.82.8` (confirmed credential-stealing malware) |
| T-01-03 | Information Disclosure | Egress to Ollama Cloud | mitigate | `model_group_settings.forward_client_headers_to_llm_api` scoped to Claude groups only — the Ollama group never receives a forwarded Anthropic bearer (D-03); behavioral confirmation deferred to Plan 02/03 |
| T-01-04 | Spoofing | Gateway exposure | accept | Bind is host-default `4000:4000` (loopback for single-user PoC); LITELLM_MASTER_KEY enforces auth. No `0.0.0.0` network exposure intended; production hardening is explicitly out of v1 scope |
| T-01-05 | Tampering | config.yaml at runtime | mitigate | config.yaml bind-mounted read-only (`:ro`) so the container cannot rewrite it |
| T-01-SC | Tampering | Container image pull | mitigate | No npm/pip/cargo installs in this phase (LiteLLM ships as a Docker image; research confirms no package legitimacy audit required). Image pin is the supply-chain control. |
</threat_model>
